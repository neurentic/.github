# Security policy

## Vulnerability reporting

Do not open a public issue to report a vulnerability.

Use one of these channels:

- GitHub Private Vulnerability Reporting, when available.
- Email: contact@neurentic.com

Include, when possible:

- Vulnerability description.
- Affected component.
- Steps to reproduce it.
- Potential impact.
- Proof of concept.
- Possible mitigation.

We will confirm receipt of the report and privately coordinate
its analysis, fix, and disclosure.
